(edited on: 2013-05-02 13:50)
Setting the minimum height of the reminders widget may be one solution to fix this issue. But the problem is, Sigmah is developed in a way that it automatically re-sizes according to the browser window and manage the scrolling internally. Since this is a desktop like application, it prevents enabling page scroll, by re-sizing the internal components according to the browser window size.

If the minimum height of the reminders widget is restricted, it will have to take the penalty from some other component ( since total height is limited in a small browser window ) most probably "Expected" widget. The best way which can think of is to set a minimum height for the whole application, where it won't re-size below that level.

But it's better to check with the layout policy of the organization for these kind of applications, before fixing the minimum height issue. This is related to the monitor size and the browser window size. Since most of the new browsers have much larger browser windows ( by removing the menu bar and other tool bars from the top ), even in small monitors browser window is much larger. Because of that this won't experience in newer browsers.
