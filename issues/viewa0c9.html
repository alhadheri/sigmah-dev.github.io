Summary0000829: Under maintenance project should not get back to "Draft" after maintenance ended
DescriptionIn some conditions, "Available & used" projects which are put "Under maintenance" get back to the "Draft" status after the maintenance is ended.
This issue can lead to destruction of fields or projects which should not be possible.
Steps To Reproduce1. create a new project model
2. set this model as "Avaiable"
3. create 2 projects from this model
4. set this project model as "Under maintenance"
5. leave the "project models" screen of the Administration
6. go back on the "project models" screen of the Administration
7. open your new "under maintenance" project model
8. leave the "under maintenance" status and click on "Save"
=> your project model is now in "Draft" status
