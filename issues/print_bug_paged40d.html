Summary0000636: Average aggregated value should not take into account empty cells
DescriptionCurrently, if an indicator having "average" as aggregation method has "4" and "20" as values, the computed "average" will be much less than "12".
Apparently, the computation takes into account empty cells as "0".
This problem is even stronger for projects covering more than one year.
Steps To Reproduce
Additional InformationIf we have: Month1/SiteA:8 ; Month1/SiteB:4 ; Month3/SiteB:12. The expected aggregated average value should be 9.

The average has to be computed first for each month independantly, and then between the different months.
So if we have (Month1/SiteA:8 ; Month1/SiteB:4 ; Month3/SiteB:12), the average for Month1 is 6 and the average for Month3 is 12, and as a consequence the global average is 9.
