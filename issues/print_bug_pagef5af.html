Summary0000259: No usage check before deleting a category
DescriptionIf a user deletes a category or a category element which is used, he won't be anymore able to open the project models or the projects which used this category or this category element.
Steps To Reproduce
Additional InformationThe best way to fix this bug is to prevent users to delete categories or category elements if they are already used in project models. In the category management screen, after the user has clicked on "Delete" for a category or a category element, if it is used by a project model he should get a warning message: "Unable to delete this category because it is used by a project model" or "Unable to delete this category element because it is used by a project model".

At best, the warning message could also inform the user about the project models and fields which uses this category or category element.
