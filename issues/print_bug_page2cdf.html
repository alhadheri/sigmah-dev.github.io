Summary0001095: Error on group creation on model with contact field without contact model restriction
DescriptionOn v2.2-rc5, in a project model, if you have a contact field without a contact model restriction, and wants afterwards to create a new layout group, you will get an error : "Unable to save group XXXX".
Actually, your group seems created normally, and according to the logs the bug seems to come from dozer (error logs attached).

If you had a contact model restriction to the field afterwards and restart the server, you won't get the error anymore.

This bug has been found while creating a new production database for an NGO. Can we trust the creation of layout groups even if we had this error, or has this error some permanent side effects that would require to restart the database from scratch ?
Steps To Reproduce1. create a new project model
2. create a layout group in the default phase => no error
3. create a contact list with restriction on "Organisation" contact, but without a specific contact model restriction
4. try then to create another layout group => you get the error
