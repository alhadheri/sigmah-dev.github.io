Summary0000132: Import and export service for categories
DescriptionAn import/export service of categories should be created in order to easily transfer categories while sharing a model accross various Sigmah instances.

A solution to deal with the issue of the global nature of a category in a Sigmah instance is to link an UUID or GUID with categories on creation.
When a category will change (adding an element or any other change), its UUID or GUID should change as well.
When a category will be transfered, if there is already another category with the same UUID or GUID, the category won't be imported, and all elements from the models attached to it will be linked to the clone category already in database.
