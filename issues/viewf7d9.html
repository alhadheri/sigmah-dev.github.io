Summary0001023: Bad imported file rows indexing
DescriptionThere are two errors on the way to index rows of imported file when using "By Row" type of importation:
- the "First row" is counted from "0" instead of "1" as it is displayed in Calc/Excel: so if the user sets "First row" value to 3, the first row in its file which will be imported is the one labelled as "4" in his Calc/Excel software
- the last row of the file is never taken into account
