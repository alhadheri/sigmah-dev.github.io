Summary0000385: Keeping test projects properly once their model no more draft
DescriptionMany users have reported the need to keep some test projects once they change their project model's status from Draft to Available.

Already, feature 0000353 (Warn user for project model status shift to "Available" if test projects linked to it) warn user about this situation.
To do the things better, when he's changing the status of the project model from Draft to Available, the administrator should get a popup with the list of all test projects linked to it and the following actions (can take the form of a drop-down list for each test project) to be runned once he validate the popup :
- delete the test project
- convert test project to real project

For each test project he wants to convert, he will have to set a separate popup prompting him separately for each project :
- a planned budget (this field will have as default value the value of the planned budget of the test project if any)
- an organizational unit (this field will have as default value the value of the organizational unit of the test project if any)
