Summary0000608: Project Activities sub-tab (GANTT diagram)
DescriptionA new sub-tab should be added in the project management: "Activities".
This sub-tab will be used to follow up the progress of the objectives, expected results, activities and tasks as in the GANTT method. This feature add a new concept to Sigmah: the task, which is division of the activity.
Additional InformationA mockup of this screen is available here: http://www.sigmah.org/issues/mockups/Activities-subtab/ [^]


Graphically, this sub-tab will look like the main tab of the GWTGantt component (available at: https://code.google.com/p/gwtgantt/ [^]) with the following modifications:
- new column: linked to logframe (to indicate that the row is also present in the logical framework)
- new column: code (code of the task, activity, etc.)
- new column: responsible (in replacement of ""resource"") with the name of the responsible of the row
- new column: Identification (in replacement of ""Task name"")
- removed columns from gwtgantt: Duration, Start, Finish, Duration & Predecessors
- new visual bar to represent the contractual planning in the last version of the project (for example, we could imagine to use a hashed bar behind the main bar to represent the contractual planning, and the main bar with transparent color over it would then represent the actual planning)
- potentially, the progress bar of the activity should not be available (because of this representation of the contractual planning)
- 2 filters should be added: a customized time period filter (enter start & end date for the view), and an team filter (filter by ""whole team"", a selection of specific positions (user profiles in Sigmah) or a selection of specific team members)

When creating/updating a new row in the Gantt chart, the following attributes have to be set:
- Type of row: Specific Objective, Expected Result, Activity, Task
- checkbox for ""do not link to logical framework"" (unchecked by default)
- Parent element: if linked to logframe and not Specific Objective, drop-down list to select the logframe parent element
- Code
- Planning (for activity & task): mandatory start & end dates, optional recurrences, plus manual settings of period when the activity/task has to be done
- Conditionnal link to other activity: START-TO-START, FINISH-TO-START and FINISH-TO-FINISH links with other activities/tasks
If the new row is linked to the logframe, the logframe will be automatically updated.

Amendment management (actually not to be taken into account for this feature directly, but more for the amendement feature)
To make this new sub-tab works as expected for their primary users which are project managers in the field, project managers should always be able to modify the content of the logframe for their daily needs, as if they always had a draft amendement to work on. In the original use case submitted by Handicap International, project managers would work only in the actual planning, and not on the contractual planning.

Right management
2 new global project privileges should be added: Update contractual planning, Update actual planning (see the Hide project/orgunit sub-tabs feature for more explanation on the concept of "global project privilege")
