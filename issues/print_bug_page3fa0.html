Summary0000278: New field type & feature: reference list
DescriptionIn some cases, users would like to be able to create lists of values in a project/orgunit which could be used as reference in other fields. The common example is the strategical axes for a country. Those axes should be able to be taken as reference in a project to specify to which axe or axes the project is linked.

This could be implemented as such:
- a new field type could be added: reference list: this field will give the opportunity to users to create a list of values (text strings)
- when creating a field of type "choice list", it could be possible to use as a source (a bit like for the categories) a field of type "reference list" either in the same project/orgunit model, or in another project/orgunit model (the logic to link fields together in separate could reuse the logic suggested in 0000213): if done so, when the user will have this "choice list" in his screen, he will have to choose one or several (depending on the multiple option of the choice list) elements created in the reference list
- if this field of type "reference list" is versioned (with the amendments), when a user want to seta value of a field which use this reference list as source of data, he will only be able to get the elements of the current validated version (or last draft if no validated version)
- a new global right should named "use archived reference values" should be added to let some users use archived reference values as source of some choice list fields, in case of entering old projects for example.
