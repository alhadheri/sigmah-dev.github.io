Summary0000532: General revision of indicators management
DescriptionSigmah v1.0 has given to the users the ability to test a materialization of a way to proceed with indicators in their organization, at the project, organizational unit and organizational levels. The question is highly complex, and during the workshops to define what's needed for version 2.0, the longed discussion started on this topic as come up to the conclusion that the functional solution to the situation could only come from a specialized 2 or even 3 days long workshop. Out of the Sigmah, the tools to monitor indicators are not yet very satifactory in many organizations, and as a consequence, the users have difficulties to express how they would like this feature to be in Sigmah. The current solution implemented in Sigmah is partially satisfactory for some users, but for all of them, it requires some improvement.

Here is the list of items to be reviewed while revising Sigmah general indicators management:
- attributes of an indicator (add baseline)
- indicators type (result indicators, impact indicators, etc.)
- process to create, monitor and enter indicators
- force the follow-up of indicators according to the type of project
- link indicators with organization's overall strategy
- new aggregrations
- organizational units indicators
- link indicators with activity progress
- extraction of indicator data
- display indicator value in dashboard projects table

The result of the 3 days workshop held in August 2013 is available in this live online document : https://docs.google.com/document/d/15I1I4_-4qQV-WsQavqrv-xzyQZrRivzwsFnR05wkbNk/edit?usp=sharing [^]
Additional InformationTwo complete scenarios have been mocked-up to illustrate this very vast feature: http://www.sigmah.org/issues/mockups/SigmahV3-Indicators/ [^]
Mockups should be revised to include those comments : http://www.sigmah.org/node/1087 [^]

To build a powerful solution for indicators, indicators could be technically "fields" like the one created in a project model. This solution should be debated.

To draw charts from indicators, the following libraries could be used:
- d3.js : http://www.d3js.org [^]
- http://www.dhis2.org/data-management [^]
- http://www.highcharts.com/ [^]
