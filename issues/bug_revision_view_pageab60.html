Another way to approach the resolution of this issue is to try to integrate an existing CaldDAV servre Java implementation as part of Sigmah. Calendars are currently under-used and a complete revision of calendar management in Sigmah is an option.
According to this wikipedia page (https://en.wikipedia.org/wiki/Comparison_of_CalDAV_and_CardDAV_implementations#Server_implementations [^] ), there are several CalDAV server java open source implementation. For example, Bedework ( https://www.apereo.org/projects/bedework/bedework-code-repository [^] ) provides a generic CalDAV server Java implementation as part of its codebase.
