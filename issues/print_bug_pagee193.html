Summary0000629: Logframe history of changes
DescriptionRight now, there is no history of modification on the logical framework.
This means that there is no solution for concurrent access problems.
An history of modifications feature should be added to each of the logframe elements, just as there is one for fields in the Phases or the Details sub-tab.
Steps To Reproduce
Additional InformationThis feature is a deep modification of the logframe.
We could use the opportunity of this change to revise completely the way logframe are designed in the code. For example, we could consider that the logframe is an hierarchical way to have a set of text paragraph fields to which indicators could be attached.

The graphical design of this feature can use the mockups done for feature 0000167.
