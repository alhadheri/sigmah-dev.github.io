Summary0000978: Custom exports of contacts
DescriptionIn order to produce easily documents, Sigmah should be able to generate exports based on templatesfrom contacts.
It is expected to have documents generated in OpenDocument Text/MS Word/OpenDocument Spreashsheet/MS Excel file format. Most expected format is MS Word.

In the Administration Contact model management screen
A new tab should be added: "Custom exports". This tab will be divided in two parts like the "Import" tab. Left side will be a table with two columns "Custom export"/"Privacy group", and with three buttons in the toolbar above: Add/Edit/Delete. And right side will be a "Contact model field"/"File template field" two columns table, with two Add/Delete buttons.
Clicking on the Add or Edit button of the "Custom exports" table will open the "Edit custom export" popup in which the admin can edit the name of the custom export, choose optionnally a single privacy group linked with this export, and must upload the expected file template. Each custom export in this list will be an hyperlink opening in the right side of the screen the list of fields mapping rules.
On the right side of the screen, each row of the table will be a mapping between a contact model field and a file template field. Those mappings can be edited by the "Map custom export fields" popup opened by clicking on a row or on the above "Add" button. This popup has two drop-down lists: left side is Contact model field, and right side is File template field.

In the contact end-user screen
If there are custom export defined for a contact model with a privacy group visible by the user, in all contacts created from this model the "Export" button is transformed for this user from a simple button into a menu button in which he see not only the "Contact Synthesis" Excel export, but also all custom exports which are visible for him. The list of exports in the menu are ordered by name ascending, and use icon corresponding to the file type.
