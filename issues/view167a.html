Summary0000270: Bad URL in user account creation email
DescriptionIn the user account creation email notice, the URL used to confirm the creation of the user account is written statically starting with "http://www.sigmah.org/". [^]
No Sigmah instance is running at this address, and the instance which send this email notice could be anywhere on the web.
So, when the user receives this email, the confirmation link sends him to a non-existing URL, and his account cannot be confirmed, so he connot log-in.
Additional InformationThe best way to fix this would be to get automatically the URL of the server form which the email is sent.
If not possible, a parameter "host.url" should be added in the sigmah.properties file.
