Summary0000633: Background transfer of files
DescriptionRight now, in context where the Internet connection is weak, it may take long period to upload an attached file into Sigmah. And while a file is uploaded, the user cannot do anything else in the software. As a consequence, it would be useful to manage a list of files being uploaded in the background.

With the offline mode, a new behavior would be possible each time a user U attachs the file:
1. a data request informs the server of the attachment of the file with all file details
2. other users can from then know that for this field, a file with all given details is currently available on user U computer and should soon be uploaded (the file field component could display this file without the hyperlink and with a text before the file name: "[Not yet uploaded]")
3. the file will be attached first in the user U local offline database
4. a background transfer of this file will automatically start once the file is stored in the the user U local offline database


Additional InformationThe background transfer of files should also be used to download files from the server into the offline local database when updating it.

Look at the following mockup to see how files should be exchanged once the offline mode will be available : http://www.sigmah.org/issues/mockups/OfflineMode/ [^]
