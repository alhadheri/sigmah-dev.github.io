Summary0000500: Export to Open Document spreadsheet format not working
DescriptionIn v1.2-rc1, both on the Global export and the Project synthesis export, the exportation fails if the *.ods format is selected.

This bug might be linked with the execution of the odftoolkit library on a Windows 7 environment. See full trace in Additional information.
Steps To ReproduceFor Global export :
1. From the projects table, click on the "Export all" button
2. Click on the "Change configuration" button
3. Change the "file type" parameter to *.ods.
4. Let the "Live data" checked by default and click on the "Export" button.
=> no file is proposed for download to the user.

And if the same procedure is done again afterwards but with Excel format chosen instead, the exportation works properly.

For Project synthesis export
1. Set in Administration / Configuration the value of the parameter "Default spreadsheet export file format" to *.ods
2. Open the Details sub-tab of a project and click on the Export button
=> no file is proposed for download to the user.

And, as for the Global export, if the same procedure is done again afterwards but with Excel format chosen instead, the exportation works properly.
