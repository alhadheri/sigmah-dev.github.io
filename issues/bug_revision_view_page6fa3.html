Summary 0000937: Outcome flexible structure
Revision 2016-09-16 17:22 by osarrat
Description Each project model should have at least one main outcome fields. This main outcome field will be one of the default field of a project model. The content of the main outcome field will be presented in the current "Logframe" sub-tab of a project which can be renamed to the value given for the outcome model used for the main outcome field. There can be several outcome fields for a project model and outcome field is a new type of field available for project, orgunit and contact models.
An outcome model is a more flexible and complete concept than the logframe. This concept is thought to match both the need to share outcomes or results between projects and to be able to monitor theory of change outcomes.
An outcome model is a set of:
- outcome model name: the name of this model, which will be used to rename the "Logframe" tab into the title provided
- presentation preference: table or graph (graph presentation will be only proposed in a future version)
- outcome levels

An outcome model may have as many outcome levels than desired. Each outcome level must have:
- a display order, ie: "1", "2"...
- a title, ie: "General objective", "Specific objective", "Expected result", "Outcome"...
- a maximum number of outcome element for this level, ie: "1" if a single outcome element for the level like a single "Specific objective" is allowed for the project implementing this outcome model, "3", "20", ...
- an optional list of fixed items to be chosen among: this list can be built according to parent level elements (for example having 3 fixed "Indicator field" outcome level (display level 2) for each of 3 fixed "Intermediate Outcome" outcome level (display level 1))
- a Yes/No parameter "Can have this outcome level indicators?"
- a Yes/No parameter "Can have this outcome level a budget?"
- a Yes/No parameter "Can have this outcome level activities?"

Some other options are possible in a future version of the feature:
- a No/Optional/Compulsory parameter "Does this outcome level provide allocation keys (%)?"
- a Yes/No parameter "Is this outcome level shared with other projects?"
- a Yes/No parameter "Should values of this outcome level be selected from external shared outcome levels?"
If "Yes" is selected for this last parameter, one or several "Outcome level data sources" must be given between the following values: "Source of funding projects with this project model", "Funded projects with this project model"
