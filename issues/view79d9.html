Summary0000463: Problem between local date and stored date in the database
DescriptionWhen an user use the software in a place with a different timezone than the database and if this user change the value of a date field, the value showed after can be inconsistent relatively to the date previously entered.
This problem is occuring only on some date field as the start date and the end date of a project.
Steps To Reproduce- Change the timezone of your PC for an upper timezone than yours
- Restart your Tomcat / Jetty if it is currently running on your PC
- Go to your Sigmah instance
- Open a project
- Open the tab named "Informations"
- Change the start date
- Save your work
- Close the project
- Re-open the same project
- Go back to the "Informations" tab
=> the starting date is one day earlier than the day previously entered
