Summary0001038: Computation field within an iterative group: pick() operator
DescriptionCurrently, it is not possible when defining a computation field set in an iterative group to include in its formula number fields part of the same iterative group.

This constraint is normal because when there are several iterations of the same number field, the computation field if out of the iterative group will need to aggregate the different values of this number field (see issue 0000886 for more details on this matter).
But in some cases, if the computation field is in the same iterative group, a simple "pick()" operator should be added as a new computation field feature to pick the value of the current iteration as part of the computation. Error messages should be send back to the administrator in cases where the pick() operator is misused: if the "picked" field is not part of the same iterative group as the computation field, or if the computation field is not at all in an iterative group.

Formula example on a "Training" iterative group for a "Training total budget" computation field part of this group: "pick(training_oif_budget_part) + pick(training_country_budget_part)" .
