Summary0000268: Hide the "None" right on Profile-PrivacyGroup link
DescriptionCurrently, if a field is linked to a privacy group, he won't be seen by the user if the user doesn't have a profil which explicilty grants him the right to See or Modify the privacy group.

If the link Profile-PrivacyGroup is charactzerized by a right "None", the users with this profile won't be able to see the content of the privacy group.
Consequently, the right "None" just duplicates the default behavior of the privacy group.

This right has a technical role, but no interest for the user. It seems useless to him and should be hidden to simplify the application for the end users.
