Summary0000255: The projet models in the choice list of Project creation window are not updated after modification on the models.
DescriptionAfter I modify the name or the state of a project model and save it, the model is saved correctly into the database. But when I get back to dashboard and try to create a new Draft Project, the model that I have already modified is not updated.

The same problem exists also after you change the state of a model from Draft to Ready,you can not see that model in the choice list when you try to create a new project.

Also, after you create a new Project Model and then try to create a new test project with this model, you can not find this model in the choice list box of ProjectCreating window.
Steps To ReproduceWhen creating a Draft Project
1. Modify the name of a project model with a "Draft" state and save it.
2. Go back to dashboard to create a new Draft Project, select a project model.
--->Problem: The name of the model you just modified is not updated !

When creating a New Project
1. Change the state of a project model from "Draft" to "Ready".
2.Go back to dashboard to create a New Project, select a project model.
--->Problem: You can not see the new "Ready" model taht you just changed.

After adding a new project model
1. Create a new project model in administration
2. Go back to dashboard to create a New Test Project,in the model choice list box, the new created project model is not found.
Additional InformationIn one word, the project model data in dashboard is not updated correctly after the modification in the Project Model Administration.
