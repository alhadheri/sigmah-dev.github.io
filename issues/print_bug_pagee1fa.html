Summary0000506: Projects visibility by attachment orgunit
DescriptionThe projects visibility rules should be changed as following:
A user can only see a set "A" of projects linked to an orgunit to which he is attached or to a child of them, and see all the projects directly linked to projects of this set A.
Simple example: a user attached to "India" can only see all projects linked to India, or to Bases in India, or to any projects linked to those projects. For the project "p-asia" attached to "South Asia" desk and linked to a project in "India", there are 2 options of visibility. If the user has no specified rights on "South Asia" desk, he will see by default the project in read-only mode, and will not be able to see any field linked to a confidentiality group to which he has no right. If the user has specified rights on "South Asia" desk, he will see the project "p-asia" according to his rights.
