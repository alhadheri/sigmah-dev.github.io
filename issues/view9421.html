Summary0000079: Manage profiles
DescriptionEach user can have one or more profiles which define the rights of this user among the application.

The rights are divided into 3 levels:
- Global permissions.
- Permissions to access to one specific org unit or one specific project.
- Permissions to read/write each element in a org unit/project.

The third level is designed with a new notion intitled "permission group". A permission group defines a code, a title and a permission (READ or WRITE). This group is linked to a profile and to a set of flexible elements.

If a permission group contained in the profile of the user matches the group of a flexible element, this user will have the right to view/edit this element.
