spM (developer)
2014-03-11 02:49

 Hi,
So I have added a patch by the name of 569.patch for this issue.

I have added two date fields for the Min and Max fields. And the Date Field is rendered in a mm/dd/YY format but is stored internally as the Linux Epoch Time in seconds using the same AdminUtil.PROP_FX_MAX_LIMIT and AdminUtil.PROP_FX_MIN_LIMIT params as it was previous, so this does not break any other functionality associated with the same.

Also, I noticed, whenever a "Text Field" was selected, there were options to select different types of Text Fields like "Paragraph", "TextArea", "Number" , "Date", and the other options like "isDecimal", "Length", "Min", "Max" also. These options were not required for all the different types of TextFields, so I have hidden the unused options for the individual TextFields.
For example, when "Paragraph" is selected, it doesnot make sense to have an option "isDecimal", so it is hidden, and similarly for Number, Date, etc.
