Summary0000816: Impossible to use project's orgunit in importation
DescriptionWhen setting a matching rule on an organisational unit field, the rule won't be taken into account both in modification and creation of a project.

In case of modification, in the "Confirm import details" popup the value entered in the cell of the project will be proposed as new value. But when cliking on the "Import" button, no modification will be made to the project and no message informs the user about this lack of modification.

In case of creation, when clicking on the "Create" button, the matching orgunit won't be proposed as default value in the create project popup.

The issue has been reproduced both in cases where only the code of the orgunit it given in the Excel file ("HQ", "AFR"...) or where it is a reproduction of the visual string display used in the "Confirm import details" popup ("ASI - Asie", "GH - Ghana", ...).
