Summary0000260: Quality framework integration in project/orgunit models
DescriptionIn order to provide a systematic follow-up of quality frameworks in the daily work of NGOs, it should be possible to decide to link a project/orgunit model to a quality framework and help administrator to track how this quality framework is used as a reference in its model.

Concretely, the following features should be added to:
- add a column "quality framework" to the screen orgunit/project models list
- add a drop-down list "quality framework" to the top part of the orgunit/project model edition screen (below the status field) with following values : "None" and the list of quality framework which have the satus "Available"
- if a quality framework is set for the model : (value different of "None")
  * show a column "quality framework" in the fields list, which will display the code of the quality framework element to which the field is linked if it is the case
  * show a drop-down list "Linked quality framework element" in the right part of the field edition popup with the following values: "None", and the list of all elements of the quality framework selected for the orgunit/project model
    + each element of this list will be displayed with 3 components: a bullet point in 3 colors (empty circle for unused quality element in the whole orgunit/project model, grey circle for unused quality element in the phase of the field, black circle for already used quality element in the phase of the field; the description of each bullet point will be provided below the drop-down list), code of the quality framework element, and title of the element
  * show a new tab "quality framework" with a table having in columns all the element of the quality framework (with code and title), in rows the containers of the orgunit/project model, and in the cell the name of the field used for the given quality framework element in the given container
Additional InformationNb: "quality criterion" is a synonym to "quality framework element"

 A quality criterion is an element of the table "quality_criterion".

In the Field form, it means concretely adding:
1. A drop-down list with the list of quality frameworks (table "quality_framework") available for the organization (if any)
2. A generic drop-down lists tree with all criterion elements (table "quality_criterion") and recursively all their children (accessible through table "quality_criterion_children"), and each drop-down list prefixed by the criterion type matching the level of the tree presented.

For example with the only quality framezork available for now, the following behavior should be added to the right of the Model Field form (both for Project Model and OrgUnit Model) :
- Quality framework : [drop-down list with 2 choices: None, or Quality COMPAS]
(After the user has chosen a choice different from None for quality framework, a second drop-down list appears with the criterion type corresponding to level 0 of this framework)
- Criterion : [drop-down list with all the criteria linked with this framework and which are not listed in the column "id_quality_criterion_child" of the table "quality_criterion_children"; for each criterion, display first its code if any finished by a dot and space (". ") and then the full name of it]
(After the user has chosen a criterion, a third drop-down list appears with the criterion type corresponding to level 1 of this framework)
- Key process : [drop-down list with all the criteria linked with this framework and which are listed in the column "id_quality_criterion_child" for "id_quality_criterion" corresponding to the just selected criterion]

A user can choose to select a link the field to a Criterion (level 0 in Quality COMPAS framework) and not to a Key Process (level 1 in Quality COMPAS framework).

I hope it makes the thing clearer for you. :)
