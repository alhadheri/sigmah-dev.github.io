Summary0000848: Confusing "NO_VALUE less than UNDEFINED" message in computation field
DescriptionIn a computation field, if a value is missing in one of the fields used for the computation, the computation field value will be marked as "NO_VALUE".

Even if there is no limit given to this computation field, the field is highlighted in red and a tool tip message informed that "Value NO_VALUE is less than the minimum value undefined". This red highlight and this error message are confusing for the user.

At best, the computation field should write "Missing value for computation" instead of "NO_VALUE", and no red highlight nor tooltip message should be assigned to the field until all required values are present.
