Issue tested by ftavin, and his test found out that this issue is linked with the way the number of lines is counted in an Excel file.
He has reproduced the test with the given file. But if he copy-pastes the content of the 9 filled rows in a new tab of the file, and renamed this new tab "Professionnels" as it must be to be imported (after deleting the old tab), then the import works. His assumption was that I had removed manually some rows in the original file to keep only 9 rows. And I have confirmed his assumption.

As a consequence, having a wrong number of filled lines, the iterator opens at some point an empty row an returns a NullPointer exception.
