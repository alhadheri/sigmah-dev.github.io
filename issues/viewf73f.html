Summary0000666: Better information when privileges prevent project core manipulation
DescriptionIf a user (like user4) which has not the privileges "Lock project core" and "Validate project core version" tries to lock or validate a project core, he will get an error message "You are not authorized to execute this action."
(test case app-122, step 2)

Additional InformationTwo solutions are possible to avoid this misleading warning message popup:
1. disable the "Lock project core" and "Validate version" buttons in this case (preferred)
2. display an information message popup "You are not authorized to execute this action." when the user clicks on those buttons..
