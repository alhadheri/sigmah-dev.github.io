Summary0000511: Category modification flexibility
DescriptionCategory should allow to have their modifications impacted directly on the project models attached to them:
- new category items can be added
- old category items can be disabled: disabled category items won't be anymore available to be selected as value for fields which haven't yet selected a value, but will remain the value for fields which have already selected them
Steps To Reproduce
Additional InformationIn the "Categories" administration screen:
- add a "Enable/Disable" button, to disable or enable a specific category item
- struck through the category name to record that the category item has been disabled
- when adding a category item, or enabling a category item which was disabled: open a Confirmation popup with this message "Do you wish to automatically add this category item as new personalized choice in all fields already using this category?"

In the "Edit field" popup, in case of a Choice list list with related category
- restrict the possible values for "Personalised choices" to category items of the related category
- only take into account as enabled category items as available category items which can be added as "Personalised choices"
