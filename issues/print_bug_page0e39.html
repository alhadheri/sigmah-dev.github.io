Summary0000367: Updating a category field breaks the field-category link
DescriptionIn the Administration Project or Orgunit Model management screen, if a user tries to update a "Choice list" field which is linked to a category, the link with the category will be lost after the 1rst update.
Steps To Reproduce1. In a draft project model, create a new field.
2. Give a name, a container, a group and an order to the field as usual.
3. Choose "Choice list" as field type.
4. On the right side of the Field creation popup, choose a "Linked category" in the drop down list (tests were conducted with a "multiple choice" choice list).
5. Save.
6. Create a test project from this project model, and check how your new choice list field appears: it should appear as expected.
7. Go back to your project model management screen, and click on the name of your new choice list field to open the edition popup.
8. Change the name of the field and click on "Save".
9. Reload the test project you've created and check the field : values from the category are no more displayed.
10. Open again the choice list field edition popup : on the right side, the "Linked category" field is disabled and set to "No linked category".
=> the link with the category has been lost during the update and cannot be recovered by the user.
