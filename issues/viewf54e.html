Summary0000630: HXL export
DescriptionIn order to be able to share the position and objective of a project, the Project Location available in the Map sub-tab should be shareable using the HXL format.

Here is the user story describing the features expected for this HXL export from this "Map" sub-tab.

User Story:

   - An organisation O is using a Sigmah set up in order to be able to share HXL 4W data, depending on the will of project manager. (we can also imagine a mode in which HXL data is automatically published for all projects)
   - A user U of this organisation O has created a new project P, with a title and an implementation period
   - In the "Map" sub-tab of his project P, U can pin on a map a point representing the project, and can choose one or several activities type among the one defined in the HXL standard.
   - Once he had fill those 2 informations, a "Share 4W data" button is enabled.
   - U clicks on the "Share 4W data" button, and the button is renamed "4W data shared publicy" and remains pressed.
Additional InformationThis feature was originally merged as part of issue 0000612, but then created as separate issue here because the HXL project couldn't provide to us the technical details in time to have this feature included in version 2.0.


Link to alpha version of the HXL standard: https://groups.google.com/forum/?fromgroups#!topic/hxlproject/oozPHu_-DUg [^]

More information on HXL: http://hxl.humanitarianresponse.info/ [^]

Blueprint of the feature for collaboration with Sahana: http://eden.sahanafoundation.org/wiki/BluePrint/HXL#SigmahtoSahana3W4Wdataexportation [^]

Discussion between Sigmah, HXL and usage of HXL data in Sahana: https://groups.google.com/forum/?hl=en#!topic/humanitarian-foss/qRUZZbsSosA [^]
