Does this problem remain even after the fix to ProjectMapper ? (http://www.sigmah.org/issues/view.php?id=165 [^])
Perhaps there are more points in the code improperly handling connections?

The problem relates to holding onto a Connections as "global variables" (singleton scope), which will eventually cause problems if (a) the database server times out inactive connections (like MySQL) or (b) sigmah is configured with connection pooling (generally a good idea) which will lead to connections being closed periodically, regardless of the database.
