Summary 0000597: Alternative currencies
Revision 2013-07-18 14:40 by osarrat
Description With the "Alternative currencies" feature, it will be possible to set up in the Administration a list of available currencies in addition to the base currency (see 0000526) for an whole organisation.
A new default field will be added to project and orgunit with budgets: "project currency" which will appear for the end user as drop-down list with all values set as "Alternative currencies" in the Administration.

When designing the budget fields of a project model (see 0000386), the administrator will have now to define:
- a set of fields, and for each field a name ("Planned budget", "Contract amount", "Spent budget", "Received amount" ;
- which field will work as planned budget (this planned budget will be used to compute the ratio of budgets in the project links) ;
- which two fields will be used to compute the budget consumption ratio.

At the end user level, a project can only use either the base currency or an alternative currency. If an alternative currency is chosen, all budget fields will be displayed twice: one version in alternative currency, and one version in base currency. Budget fields in alternatice currency will then be highlighted, budget fields in euros will be presented as second importance and a persitent warning message will inform the end user that all amount in base currency are actually converted and should be taken with caution.
If the base currency is chosen for the project, the budget component will only show the budget fields in one version: associated with the base currency.

The warning message that will inform the end user that all amount in base currency are actually converted" should also be added into the linking project pop-up and screen part to avoid misunderstanding.
