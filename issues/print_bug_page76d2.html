Summary0000004: Setup maven to generate XML schema for Report model classes
DescriptionAI's report generation mechanism uses an XML binding to define and store report definitions. Currently, the XML schema is not documented outside of the source code.

We need to setup JAXB to generate an XSD schema from the java classes which can then be used to generate user-friendly-ish docs for writing report definitions, at least until the UX for designing reports is finished.

Steps To Reproduce
Additional InformationJAXB models are in org/sigmah/shared/report/model

See http://www.jumpingbean.co.za/blog/maven-jaxb-schemagen-errors-pom.xml [^] for some general orientation-- the JAXB plugin has been updated since this writing so there is no need to define the dependencies.

The maven jaxb-schemagen plugin is braindead, doesn't add your source to the classpath, so the schema generator can't load the typeadapters.

This should still be added to the POM, but embedded as an ant script using the maven-ant plugin and the xjc jar as a dependency.
