Thanks nikita for solving this issue!
But actually, the expression of issue has been ambiguous, and this issue reported in 2011 on v0.9 has bee fixed between 2011 and today.
The title "When "Unavailable" a project model can not be "Available" again" is ambiguous: it is not the objective of the issue in this case but the reported problem. So what you tried to fix (banning a change back to "Available" when a project model is "Unavailable") is actually not an issue. What was an issue on v0.9 was that it was not possible as expected to change back to "Available" a project model which is "Unavailable".

So I close this issue which has already been fixed.
