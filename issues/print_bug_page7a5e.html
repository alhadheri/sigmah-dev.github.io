Summary0000792: Add projects count to project types in Dashboard Projects table
DescriptionCurrently, there is a behavior which is not very intuitive for new users. When a new user creates a "source of funding" or "local parnter" project in the Dashboard, the project will be added in the Dashboard projects table but won't be most probably visible because the default grouping of the table is by "in-house project". As a consequence, he may believe that his project has not been created, and will try to create it a second time... (behavior checked on demo.sigmah.org)

To fix this, adding a projects count to each of the 3 project types in the top banner of the Dashboard projects table would be a solution.
This solution may look like:
"Group by: [x]In-house project(23) []Source of Funding project(37) []Local Partner project(19)"
Each time a project will be created, the projects counters will be increased. The counters will of course be refreshed each time the table is fully refreshed.
