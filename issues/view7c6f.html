Summary0000252: Project links display is not correct when moving from a project tab to another
DescriptionIn some cases, links are not correctly displayed, percentages are inverted and the link project name displayed is the current project name instead.
Steps To ReproduceThe following scenarios have been reproduced both in the following environment :
- on a local instance running on Jetty 7.4.2 with PostgreSQL 9.1
- with Mozilla Firefox 7.0.1
- in fresh organization created on an existing database (which contains already other organization)

Prerequisites :
- create 3 project models : in-house project model, funding project model, local partner project model (no need to add any field to any of those project models)
- make those 3 models "available" (but the bug has also been reproduced with project model "draft"
- create 2 projects from each one of those project model : "ih1" and "ih2" for in-house projects, "fp1" and "fp2" for funding projects, and "lp1" and "lp2" for local partner projects.


First scenario :
- Open "ih1".
- Create a new link for Funding project with "fp1"
- Create a new link for Funded project with "lp1"
- Go to another project tab
- Return to project dashboard
--> Links are not correctly displayed, percentages are inverted and project names take the name of "ih1" instead

Second scenario :
- Open "ih2"
- Create a new link for Funding project with "fp2"
- Create a new link for Funded project with "lp2"
- Modify one link percentage
- Go to another project tab
- Return to project dashboard
--> One of the links is not correctly displayed
