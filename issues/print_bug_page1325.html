Summary0000549: No visibility control on "Reports & documents" sub-tab
DescriptionEven if a Files attachement field is not visible to a user due to unsufficient privileges on its confidentiality group, the files attached to this field remain visible for this user in the "Reports & documents" sub-tab.
The content of the invisible fields to a user should be invisible to him everywhere.
Steps To Reproduce (i) set a privacy group to a "file list" field. Add one or several files to this "file list" field.
(ii) open the project with an user having not the right to see the privacy group
(iii) on the "management board, nothing is visible (OK) but on the "report and documents" sub tab, all the files are visible and editable ("add file" button available if the user as the right to "dit project") despite their privacy group.



Can also be tested with integration test case http://code.google.com/p/sigma-h/wiki/TestScenario4Rights#R1 [^]
