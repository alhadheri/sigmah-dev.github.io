Summary 0000883: Document field: improved former "file list" with hyperlinks
Revision 2016-08-11 12:45 by osarrat
Description Users wish to add hyperlinks in Sigmah.

The proposed solution is to extend the existing "File list" field in a new "Document field" able to handle hyperlinks.

In the administration:
- "Files list" field type should be renamed "Documents list"
- Documents list should have two new specific properties option as checkbox: [] handle files, [] handle hyperlinks (at least one of the checkbox should be checked, and "handle files" checkbox should be checked by default)

In the project/orgunit page: the documents list field should look like the given mockup:
- if it handles both files and hyperlinks, it should have an "Add file" and "Add hyperlink" buttons
- each of this button is now a menu button in which the user can choose to add a new file/hyperlink, or edit an existing file/hyperlink
- a few pixels of white blank should be added below the document field so that is several documents field are following one another, they are not stick to one another


This issue has also an impact on the "Reports & documents" sub-tab where:
- a new "Add hyperlink" button should be added
- each menu item of the "Add a file" and "Add an hyperlink" menus should open a sub-menu with a "Add a new file in this field" option and several options named as file names in this field below a "Update existing file in this field".
