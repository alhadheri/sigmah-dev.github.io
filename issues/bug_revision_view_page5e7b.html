Revision 2016-09-13 19:50 by osarrat
Description In order to manage data frameworks (see 0000860), Sigmah lacks a screen to edit such an object.

In this screen, the user will be able to:
- edit the arborescent structure of the data framework: create, edit & delete groups of elements, and create, edit & delete data elements
- set the status of this framework (like in models) : "Draft", "Available", and "Unavailable"
- [optional] set the languages in which this data framework will be available, and set the translations of each element in the desired language (see child issue 0000121)
- attach a data framework content export plugin file
- set a URL for documentation on each of the element of the framework (this URL could be used by the user to describe more in depth what is this element of the data framework)
- set a global mapping rule for some data elements

This screen will be very similar to the screen for editing project models: a top part for editing the data framework title, status, and to attach the data framework content export plugin file.
The bottom part of this screen will have several tabs:
- "Structure" where groups of elements and data elements can be modified, in a similar manner than fields in project model.
- "Implementations" where the administrator can see how each project model has implemented or not the data framework: it will be a table filled which each available project model, and with 3 columns: "Project model" (each project model name will be an hyperlink to the project model), "Implemented" (Implemented/No/Not Yet) and "Comment for non-implementation".

The "group of elements" and "data element" edition popups will be quite similar to the field edition popup. In this popup, in addition to specifiy the details of group of elements or the data element, it will be optionnaly possible to set the URL for documentation, and set a global mapping rule for data elements. Three options are possible for this global mapping rule: either "No rule" (by default), "Export generation date/time" or "Constant". If the admin selects the "Constant" option (might be represented as a radiobutton), he will have to filled an additional text field which will be the constant.
