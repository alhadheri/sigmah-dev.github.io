Summary0001039: Global export fails if contains any iterative group field
DescriptionOn v2.2-rc2, if you include a field belonging to an iterative group as part of the project global export, the global export will fail: it will produce a file not reproduced by MS Excel or LibreOffice Calc.
Steps To Reproduce1. Create a project model with an iterative group in a phase, and set the project model as available.
2. Create a project out of this project model.
3. Create an iteration of the iterative group and fill one of its field.
4. Go back on the dashboard, and click on the "Export all" button.
5. Click on the "Change configuration" button, and add several fields of the new project model as part of the global export, but yet no field from the iterative group.
6. Click on the "export Live data" button.
=> you get a global export file
7. Click again on the "Export all" button and then "Change configuration".
8. Add any field from the iterative group.
9. Click again on the "export Live data" button.
=> the global export file you get cannot be opened and contains a mistake
