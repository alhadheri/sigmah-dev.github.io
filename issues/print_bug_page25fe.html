(0001393)
somzzz   
2014-03-13 00:22   
Discovered and fixed the following bug:
Navigating away from the Administration tab saves the currently opened page (example: "Administration/0" or "Administration/1/889/All_Tabs"), but returing to it does not properly configure the navigation panel if the page you left is a Administration/Tab/Subtab. Unlike the usual behaviour for a tab/subtab, when the navigation panel shows a label without hyperlink, it now shows a clickable hyperlink. Clicking it opens the list of models.

This behaviour is not natural for the existing code and is caused by an untreated case and a dependency between the loading of distinct visual content (the panel and the main view) and the present navigation chain.

Details about how this bug was fixed and integrated in the patch for issue 599 are provided in the readme file attached to the patch submission.

Steps to reproduce ( for revision 1559 ):
- open Administration/2/pick_a_project or Administration/3/pick_an_organization
- return to the dashboard without closing Administration
- return to the opened Administration tab
- the entire navigation panel is clickable
- click Administration/2 or Administration/3, return to the lists of models
