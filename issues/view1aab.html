Summary0000457: Installing/Building Sigmah on Postgres using current script fails for certain country locations
DescriptionIf user(ngo) try to install sigmah on postgres using current postgres sql dump , then the process fails at New Organization creation script(Filename=newOrganizationLaunchScript-1.1.1+.postgresql.sql).

Same applies to developer while setting up the environment.



Steps To ReproduceExecute scripts using PHPPGadmin or execute directly to postgres.

1) First execute minimum data kit.
2) Execute new Organization launch script. While specifying ISO 2 letter country code as referenced in same SQL (http://en.wikipedia.org/wiki/ISO_3166-1_alpha-2#Officially_assigned_code_elements [^]), Use it for India for example.
3)Upon execution, you will find that transaction fails and process rollbacks.

Applies also to country :
1) Brunei Darussalam
2) China
3) Guinea
4) Honduras
5) Mongolia
6) Netherlands Antilles
7) Pitcair
8) Saint Kitts And Nevis
9) Senegal
10) Tunisia.
11) VietNam
Additional InformationCreation of user_unit_profiles fails in OrganizationlaunchScritp with NULLVALUES.

Further IF we trace , we find This has failed beacause actually user_unit was NULL.

Then going further you find, even UserLogin creation has failed.

Thus solve all nestings of this sql file and then you will find that it fails atline 106 where theres is iso code comparison i.e. c.ISO2='§HeadquartersCountryCode§'.

Then after checking you will conclude, it because of dependency of Minimumdatakit and especially file name called 1-actinfo-data-withoutUser.postgresql.
