Summary0000924: Importation frameworks for Contact models
DescriptionIn order to fullfill the requirement to map a file template to a contact model, an "Importation frameworks" tab is necessary in the Contact model edition screen.

The behavior of the "Import" feature should evolved to take into account contacts as a new kind of entity which can be created or update during an import.
Additional InformationSuggestion of a functional design to reply to this need

General importation process:
1. From anywhere the "Import" button is clicked (global menu, contacts table, contact list field), the same "Import" popup will open.

2. Once opened, the "Import" popup will ask first only to select the Importation framework in a drop-down list of all Importation frameworks available. If a user has not the right to import contact ("Import contacts" global privilege not checked), or not the the right to import other entities than contacts ("IMPORT_BUTTON" global privilege not checked), then the Importation framework drop-down list will be filtered accordingly: only importation frameworks which are linked to models of entities to which the user can import will be listed.

3. Once the framework is selected, if it is linked to models of different entities (ie: project and orgunit, contact and project, etc.), a list of 3 radiobuttons project/organisational unit/contact will be displayed prefixed by the label "Importing to:". The radiobuttons will be enabled/disabled according the right of the user to import to those kind of objects. And for each radiobutton, if the entity is linked to the framework and if the user has the right to import to that entity, the radiobutton will be checked by default.

4. At the same time the list of 3 radiobuttons will be displayed, the rest of the popup will be displayed, but with the label "Create a new project when no match is found" replaced by "Create a new entity when no match is found".

5. If the importation will import contacts, the "Verify each match" mode will be disabled, and the "Unlock the project core if necessary" checkbox is disabled and unchecked. Moreover, after clicking on the "Import" button, the imported file will be completely parsed and a deduplication check for each file entry will be made.

6. If the importation will import contacts, at the end of the importation a final "Summary of the contact importation" popup will open with the following informations:
* "File entries summary" (label)
  - "Total number of file entries"
  - "Number of contacts created"
  - "Number of contacts updated"

Importing from contact list field specific importation process:
When importing contacts from a contact list field, the importation must not only create contacts but also attach contacts in this list. As a consequence, the General importation process is modified as follows from step 2 onward:

2. Once opened, the "Import" popup will ask first only to select the Importation framework in a drop-down list of all Importation frameworks available. The Importation framework drop-down list content will be filtered accordingly: only importation frameworks which are linked to contact models matching the constraints of the contact list field will be listed (constraints in terms of individual/organisation types, or specific contact model restrictions. If the contact list only accepts a single contact model, the "Importation framework" drop-down list will be set to this unique possible value, will be disabled and the popup will thus directly display also content displayed from step 3 of this process.

3. Unlike the General importation process step 3, no entities radiobuttons can be displayed when importing into a contact list.

4. Similar to General importation process step 4.

5. Similar to General importation process step 5.

6. At the end of the importation, a final "Summary of the contact importation" popup will open with the following informations:
* "File entries summary" (label)
  - "Total number of file entries"
  - "Number of contacts created"
  - "Number of contacts updated"
* "Contact list field modification summary" (label)
  - "Number of contacts added to the list"
  - "Number of contacts not added because exceeding the number of contact's limit of the field"
  - "Number of potential not added because not matching the 'CONTACT_MODEL_CHECKBOX_FIELD_NAME' constraint"
