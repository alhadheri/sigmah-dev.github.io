tmiette   
2011-09-30 09:40   
The images service works with tomcat on our demo server. It's not a bug, it's a configuration mistake.

repository.name : this is the root directory of the repository where all external files are stored by Sigmah.

repository.images : this is a SUB directory in the root directory where the logos are stored (no need to add any slash).

So, according to your configuration, your directories tree must be something like this :

# sigmah.properties
repository.name=C:\\sigmah-repo
repository.images=sigmah-logo

# your drive
C:
|_sigmah-repo
  |_sigmah-logo
