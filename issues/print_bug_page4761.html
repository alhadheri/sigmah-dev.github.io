Summary0001048: Missing ban on creation of contact without main orgunit
DescriptionOn v2.2-rc2, it is possible to create a contact without a parent organisational unit.
This generates an "Unexpected error", the contact won't be displayed in the Dashboard contacts table, but it will be possible to select it in a contact list from a project or another contact. And if it is selected from a contact list in a contact, it will be possible to open it from the "Relations" sub-tab. Then, this contact created without a parent orgunit will appear without any field.

Creating a contact without a main parent orgunit should be banned.
