Summary0000372: Impossible to change the project model status once no more "Draft"
DescriptionIn the screen to manage a project model, after the user has changed the status of a project model from "Draft" to another value (either "Available" or "Unavailable"), the "status" field is disabled, and he cannot anymore change it.
The "status" field is also disabled if the project model is in the "Used" status.

This field should never be disabled.

osarrat (administrator)
2011-09-21 10:43

 1. => You're right. Once no more "Draft", the content of the model (but not its status) cannot be modified.

2. => "Unavailable" means "no more visible to the end user". But an administrator can decide to make a model no more visible... and then change his mind and decide to make it visible once again. The only change which is not allowed is coming back to Draft once Used.


To make it clearer, technically, there should be 2 informations to track : status asked by user, and "used" status (which is just True/False).
Once created, a model is always Draft.
If Draft, user can only set model status to Available.
If Available, user can set model status back to Draft or to Unavailable.
If Available, user can create a project from the model which then makes the model status appearing as Used. The "used" internal status will then never be changed and remains "true".
If Used, user can only set model status to Unavailable.
If Unavailable and never "used", user can set model status back to "Available"
If Unavailable and already "used", user can set model status back to "Used".

edited on: 2011-09-26 16:43

 Some improvements are still needed for this bug fix. The 2 main issues remaining are:
- user should not be able to choose the status "Used" for a model.
- instead of using error messages, it would be better to prevent users from selecting not allowed values for the status

To make things clearer, I suggest to rename the status "Used" into "Available & used".
Here is the full exact behavior expected for model status management (both for project or orgunit models):
1. Model status is Draft
Status drop-down list has following available values: "Draft", "Available", "Unavailable".

2. Model status is Available :
Status drop-down list has following available values: "Draft", "Available", "Unavailable".

3. Model status is "Available & used" : (only because a project or orgunit has been created out of the model)
Status drop-down list has following available values: "Available & used", "Unavailable".

4.a. Model status is Unavailable, and has never been Used :
Status drop-down list has following available values: "Available" and "Unavailable".

4.b. Model status is Unavailable, and has already been Used :
Status drop-down list has following available values: "Available & used" and "Unavailable".

There should be no warning message for any change of status if only those values are presented.
