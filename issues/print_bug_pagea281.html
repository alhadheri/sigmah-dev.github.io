Summary0001093: Unable to export or duplicate a project model having a contact field
DescriptionOn the attached database, it is not possible to export the project model "Internal project test", whereas it is possible to export other project models.
After some testing, I found out that if a project model has a contact field, it won't be possible to export or duplicate it.

When you try to export that project model, you get a screen "Export functionality is only available when online.".
-
In order to understand why we get this bug, man should run a v2.2-rc5 Sigmah instance in debug mode with the attached dump as database to follow the code running and understand what is happening.
Steps To ReproduceOn v2.2-rc5:
1. create a project model, and add 2 numeric fields which you use as planned and spent parts of the budget ratio field
2. export this draft project model => it works
3. duplicate this draft project model => it works
(3'. delete your project model duplicate just created)
4. add a contact field to this project model
5. export this draft project model => it doesn't work
6. duplicate this draft project model => it doesn't work
7. delete this contact field from this project model
8. export again this draft project model => it works
9. duplicate again this draft project model => it works
