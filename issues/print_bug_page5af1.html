Summary0001004: Unexpected error if user has not right to edit matched project on importation
DescriptionOn v2.2-rc1, if a user imports data which will edit a project which he doesn't have the right to edit, he will get an "Unexpected Error".

The log file gives more detail: "The project PPPPPPPPPP is not editable by the user UUU" (see full example below).

Trying to edit by importation a project on which a user doesn't have sufficient privilege should not generate an unexpected error, but should be recorded as a special case in the importation result: "Project found but not updated because user USER_NAME does not have sufficient privilege for this update".

The behavior is not the expected behavior. If importing data on a project a user has no modification privilege (because lack of Global Privilege "Modify Project"), projects will be considered as "Project not found" by the importer (see latest attached screenshot).
