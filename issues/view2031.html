Summary0001024: Impossible to import with some special formatting of the content
DescriptionTwo files having exactly the same content won't have the same behavior in importation: one without any formatting will be imported while the other will generate an Unexpected error.
Steps To Reproduce1. Create an importation framework with those parameters: (File format: Excel / Projects to import per file: By row / First row: 2 / Sheet name: Export to Sigmah)
2. Add a unique variable: (Column: A / Name: PID)
3. Attach this importation framework to an Available or Used project model, and map the "PID" variable to project Code.
4. Try to import the "Working" Excel file in Mass mode with the "Create a new project when no match is found" checkbox checked
=> it works
5. Try to import the "NotWorking" Excel files with the same options
=> it does not work (Unexpected error with log information below)

You can repeat the two last steps in any order, you will always get the same result.
