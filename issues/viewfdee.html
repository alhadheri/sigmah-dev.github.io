Summary0000415: Impossible to see Indicator Sites from other users
DescriptionIf a user 1 has created site for a specific project, a user 2 with same rights and attached to same orgunit as user 1 won't be able to see those sites.
Steps To ReproduceOn a database with one empty project and at least 2 users with same profile and attached to same orgunit.
With user 1:
1. create a quantitative indicator
2. create 2 sites
=> the quantitative indicator is available for data entry in the "Indicator Data Entry" sub-tab of the project.

With user 2:
1. open the project where user 1 has just created the indicator and the sites
2. open the "Indicator Data Entry" sub-tab => it is impossible to enter indicator values
3. open the "Sites" tab in the "Indicator Management" sub-tab => it is empty

osarrat (administrator)
2011-11-10 13:16

 I found that the following query enabled all end-users of one organization to enter indicator values in "Indicator Data Entry". With those permissions given to users, they still cannot see sites they haven't created. It shows also that the issue is partly linked with a bad management of ActivityInfo permission system inside Sigmah permission system. I found out actually that the table "userpermission" was totally empty.

Could you validate that the following query would be a good quick SQL fix?


INSERT INTO userpermission (userpermissionid, allowdesign, allowedit, alloweditall, allowmanageallusers, allowmanageusers,
 allowview, allowviewall, databaseid, partnerid, userid)
SELECT nextval('hibernate_sequence'), TRUE, TRUE, TRUE, FALSE, FALSE, TRUE, TRUE, udb.databaseid, MIN(uu.id_org_unit), ul.userid
FROM userlogin ul
    inner join user_unit uu on (uu.id_user = ul.userid),
     userdatabase udb
    inner join userlogin o_ul on (udb.owneruserid = o_ul.userid)
WHERE ul.id_organization = 1 and o_ul.id_organization = ul.id_organization
GROUP BY ul.userid, udb.databaseid
ORDER BY ul.userid, udb.databaseid;
