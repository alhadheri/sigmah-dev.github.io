Summary 0000260: Quality framework integration in project/orgunit models
Revision 2011-10-04 13:01 by osarrat
Description In order to provide a systematic follow-up of quality frameworks in the daily work of NGOs, it should be possible to decide to link a project/orgunit model to a quality framework and help administrator to track how this quality framework is used as a reference in its model.

Concretely, the following features should be added to:
- add a column "quality framework" to the screen orgunit/project models list
- add a drop-down list "quality framework" to the top part of the orgunit/project model edition screen (below the status field) with following values : "None" and the list of quality framework which have the satus "Available"
- if a quality framework is set for the model : (value different of "None")
  * show a column "quality framework" in the fields list, which will display the code of the quality framework element to which the field is linked if it is the case
  * show a drop-down list "Linked quality framework element" in the right part of the field edition popup with the following values: "None", and the list of all elements of the quality framework selected for the orgunit/project model
    + each element of this list will be displayed with 3 components: a bullet point in 3 colors (empty circle for unused quality element in the whole orgunit/project model, grey circle for unused quality element in the phase of the field, black circle for already used quality element in the phase of the field; the description of each bullet point will be provided below the drop-down list), code of the quality framework element, and title of the element
  * show a new tab "quality framework" with a table having in columns all the element of the quality framework (with code and title), in rows the containers of the orgunit/project model, and in the cell the name of the field used for the given quality framework element in the given container
