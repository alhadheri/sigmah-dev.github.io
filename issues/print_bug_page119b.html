Summary0000524: Dashboard projects table indentation
DescriptionA new ordering should be made in the Dashboard Projects table.
Here are the new rules:
- Several levels of indentation are now possible in the Projects table if projects of the type used for regrouping are linked together. The indentation will work with those rules:
* only projects of the type used for regrouping will be expanded further (ie: in a list regrouped by Own project, only own projects can be expanded further if "children" of another own project)
* as a consequence, in order to avoid that projects of the type used as regrouping are listed twice, a pair of projects of the same type will be listed only once in the form where the project funding the other will appear on top of it (ie: if own project A funds own project B, own project A will appear on item at the root of the list in the projects table if the table is regrouped by own project, and B will appear as a direct child of project A, and not appear as a project at a root position in the list with A as a child)
- When expanding a project, the project listed below are first odered by type (Funding first, Own project second, Local Partner third)
- For the rest, the behavior of the Projects table will be the same.
