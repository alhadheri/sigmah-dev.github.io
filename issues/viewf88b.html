Summary0000501: Group iterations
DescriptionA new feature should be added to project/orgunit groups: group iteration.
A new option will be added to groups to set if iterations can be made on this group.
If iterations are made on this group, it will be possible for the end user to create different iteration of all the fields included in this group. Visually speaking, you can look at the draft mock-up attached to this issue.
A new global privilege: ""create new group iteration"" should be added.
Management of the mandatory fields:
- a graphical solution should be found to distinguish the group iterations for which all the included mandatory fields have been filled from those for which some fields are still waiting some content
- the included mandatory fields will be considered as filled at the phase level only if they are filled for all iterations of the group

When creating a new iteration, the end user will have the possibility to change its default name which will be a simple number. This iteration name can be rename with a contextual menu available on the right of the iteration name with a small triangle pointing down (just as in the logframe).
