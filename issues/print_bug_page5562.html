Summary0001113: Details missing for out of user's orgunit scope's contacts attached to a user's project
DescriptionOn v2.3-rc1, for a user having a limited "visible orgunits' scope", a contact out of this scope but attached to a project whose team contains the user will not be visible in the Main Dashboard Contacts table, and its details won't be displayed if the user manages to open it from the project contact list field.
Steps To Reproduce1. create a user U attached to a lower orgunit O (not the HQ) as main orgunit
2. create a project P attached to this lower orgunit O
3. add this user U in the team of this project P
4. add a contact C from a higher orgunit than O in a project P list of contacts field
5. close the current session and reopen a session with user U
=> in user U dashboard, the contact C won't be displayed in the contacts table
6. open project P, and look for the list of contacts field where the contact C has been added
=> the contact C is correctly visible in the list of contacts
7. click on contact C
=> a tab opens for contact C, but no information is available on this contact
