Summary 0000957: Unable to build budget ratio migration Java tool
Revision 2016-09-27 23:55 by osarrat
Description On v2.2-rc1, the expected missing automatic migration behavior is missing on the new Revise budget default field design ( 0000864 ) feature.
As a consequence, loading the projects table on the dashboard creates many error in the log.

Potentially, this also lead to the impossibility to export all files, since exporting was actually working on v2.2-beta3 before integration of 0000864.
Revision 2016-10-12 09:58 by osarrat
Description On v2.2-rc1, the expected missing automatic migration behavior is developed through a Java tool available on https://github.com/sigmah-dev/sigmah/tree/master/scripts/migration_budget_field [^]
But the pom.xml of this small Java project is almost empty, and as a consequence it is impossible to build it. No documentation is either provided to build it.

So it is now impossible to migrate budget ratio from v2.1 instances. And as a consequence, loading the projects table on the dashboard creates many error in the log. Potentially, this also lead to the impossibility to export all files, since exporting was actually working on v2.2-beta3 before integration of 0000864.
