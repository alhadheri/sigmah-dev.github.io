Summary0000213: Custom exports
DescriptionIn order to produce easily documents required in the daily life of a project, Sigmah shouldbe able to generate exports based on templates.
It is expected to have documents generated in OpenDocument Text/MS Word/OpenDocument Spreashsheet/MS Excel file format. Most expected format is MS Word.

In the Administration Project model management screen
A new tab should be added: "Custom exports". This tab will be divided in two parts like the "Import" tab. Left side will be a table with two columns "Custom export"/"Privacy group", and with three buttons in the toolbar above: Add/Edit/Delete. And right side will be a "Project model field"/"File template field" two columns table, with two Add/Delete buttons.
Clicking on the Add or Edit button of the "Custom exports" table will open the "Edit custom export" popup in which the admin can edit the name of the custom export, choose optionnally a single privacy group linked with this export, and must upload the expected file template. Each custom export in this list will be an hyperlink opening in the right side of the screen the list of fields mapping rules.
On the right side of the screen, each row of the table will be a mapping between a project model field and a file template field. Those mappings can be edited by the "Map custom export fields" popup opened by clicking on a row or on the above "Add" button. This popup has two drop-down lists: left side is Project model field, and right side is File template field.

In the project end-user screen
If there are custom export defined for a project model with a privacy group visible by the user, in all projects created from this model the "Export" button is transformed for this user from a simple button into a menu button in which he see not only the "Project Synthesis" Excel export, but also all custom exports which are visible for him. The list of exports in the menu are ordered by name ascending, and use icon corresponding to the file type.
Steps To Reproduce
Additional InformationOld broader description of the feature (only given for information)

For a project model or an organizational unit model, a user can attach several custom exports.

Each custom export has:
o A name (ie: “Concept note”)
o A location for the button to create the custom export (comprehensive list of the location for the Export drop-down menu buttons is: each phase, the “details” sub-tab, each opened report (the button will be then linked to the report flexible element in the model))
o A template (could be for example a LibreOffice Write or MS Word form, but other file types should be also available like LibreOffice Calc, MS Excel, csv, etc.)
o A mapping between the fields of the template and the fields of the linked project or organization unit model
o It would be also possible to link the fields of the template to fields of projects/orgunits linked to the project/orgunit generated from the project/orgunit model

If the custom export is linked to a project model, a template field could be linked to :
o a field of a project model used as funding source
o an information of a project link to a project model used as funding source
o a field of a project model used as funded project
o an information of a project link to a project model used as funded project
o a field of an orgunit model used as parent orgunit of the project
o indicators of the project (see below for more information)
o logical framework of the project (see below for more information)


If the custom export is linked to an orgunit model, a template field could be linked to :
o a field of an orgunit model used as parent orgunit of the orgunit
o indicators of the orgunit(see below for more information)

If several projects match the project model of the linked field, the user will have to choose which project he want to use in his export.


__Linking indicators to orgunit / project template__
The solution to link indicators to an orgunit / project template could be discussed when this issue will be implemented. A solution could be:
- in the custom export management screen, a field of the template could be linked to an element called "indicators block" (and all indicators will be inserted at the position of this field in the template)
- when managing the link with the "indicators block", the administrator has to define at least two text styles available in the template as style available for presenting (i) "group of indicators" and (ii) "indicators" themselves (more text styles could be use to present differently the various levels of groups of indicators)
- a unique indicators presentation structure will be available at the beginning, but more should became available in the future (we can already imagine that we will need in a close future both a full text and a table presentation structure for this "indicators block")



__Linking logical framework to project template__
The philosophy to implement the presentation of the logical framework in the template could be the same as the philosophy for indicators.

In the future, it would be possible to produce custom exports which will integrate several linked projects/orgunits.
