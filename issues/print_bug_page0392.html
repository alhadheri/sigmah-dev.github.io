Summary0001016: Very large contact database management
DescriptionIn order to be able to manage very large contact database (more than 5000 contacts), several improvements need to be done in Sigmah:
- ensure correct performance (less than 2s) for creating, modifying, displaying a contact
- ensure correct performance (less than 5s) for checking for contact duplicates
- ensure correct performance (less than 30s) for loading the full contact table on the dashboard
- replace all drop-down lists of contacts which might try to have in available contacts the whole contact database or a very large portion of it(see additional information for more details on this matter)
Steps To Reproduce
Additional InformationExhaustive list of graphical interface components which should be replaced:
- "Add" a new contact drop-down list in the contact list field
- "E-mail" contact drop-down list in the "Create user from a contact" popup
- "Parent organisation" contact drop-down list in each contact page

All those drop-down lists should be replaced by a text field next to a "Search and select" button. According to the content of the text field, the "Search and select" button will open a popup with different search results:
- if the content of the text field is an email address, the popup will be filled with all contacts matching the email address
- if the content of the text field is a number, the popup will be filled with the contact having this number as "Contact identifier", and all contacts which might have this number as name
- in all other cases, the popup will be filled with all contacts having approximately the content of the text field as name
Of course, the result of the popup will be filtered according to the purpose of the field: limitation on the type of contact (individual/organisation) or the model of contact will be respected.
The "Search and select" popup will display the results in the form of a table similar to the "Possible duplicates found" popup, with a "Select" button at the end of each row. A "Cancel" button will be available at the bottom.
