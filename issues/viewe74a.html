Summary0001070: Impossible to view a project "Details" sub-tab if another project is in old core version viewing mode
DescriptionOn v2.2-rc3, after requesting to display a former project core version in a project, each time you will open the "Details" sub-tab of another project, the tab you would have opened will be renamed to the project code of the project you have requested to display a former project core version.

And on v2.1, there was already a client-side unexpected error when following those reproduction conditions.
Steps To Reproduce1. open a project (like IF1294 on the demo database)
2. lock project core and validate a new project core version
3. modify one of the fields of the project core and Save
4. request now to display the project core version you have created in step 2
5. visit the 'Details" sub-tab of another project
=> you get the strange behavior
Additional InformationThis issue is a remaining part of issue 0001045.

And it has been tested and confirmed that this issue cannot be reproduced on v2.2.
