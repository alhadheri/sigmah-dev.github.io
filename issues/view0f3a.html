Summary0000691: Impossible to select projects part of offline data base
DescriptionOn v2.0-rc1, when you synchronize your offline data, all the projects are synchronized. Normally, only the favorite projects must be synchronized. This is required because a user should have the possibility to reduce to the bare minimum the volume of data to synchronize.
Steps To Reproduce1. Destroy your offline file & data bases
2. Open a not favorite project
3. Switch offline
4. Go back to the Dashboard and try to open a favorite project, and then another not-favorite project => you cannot open them
5. Close all opened projects, and open back your initial not favorite project
6. Switch back online and "Update offline data base"
7. Switch back offline and try to open another not favorite project
=> you can open it when it shouldn't be possible

rcalabro (administrator)
2015-04-14 11:59

 It is due to 2 facts :
- "auto learning" feature of Sigmah: each visited project and downloaded file is cached in the offline mode, favorite or not.
- related projects: every related project is cached since it is required to load the favorite project.

Opening a not favorite and not related project is not possible (verified on Sigmah 2 rc1).

Removing the "auto learning" feature will require more data to synchronize projects since they will not be cached at all.

 Issue reopened because there is an inconsistency on the projects available offline:
- the offline data base store both the favorite projects and all projects directly linked to it
- the "Transfer files" popup propose only to download files coming from the favorite projects, and not from projects related to favorite projects

To fix this inconsistency, the solution easier to understand for the end user would be to limit projects available offline to favorite projects, and not to include projects related to them.

rcalabro, you write that "every related project is cached since it is required to load the favorite project". To understand better, how can you then load related projects without caching the projects related to related projects ?
