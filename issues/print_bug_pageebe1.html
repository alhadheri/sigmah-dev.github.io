Summary0000374: Remove unnecessary uniqueness check on report model section names
DescriptionAs reported in the attached User report, some users have trouble with the feature which prevents the entering of duplicates of report model section names.

The general philosophy of the administration is to let the maximum freedom to administrators in their configuration. There is no functional reason to keep this control, and it can be fully removed.

A user should be allowed to enter twice the same report model section name, for example when he has 2 sub-sections with same name which belongs to 2 different parent sections.
Steps To Reproduce
Additional InformationThere is no unicity constraint on the report model section name in database.
