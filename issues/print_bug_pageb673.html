Summary0000281: Automic closure for todo reminders
DescriptionSome actions of users need to done on a regular basis. The reminders could be used to help users to manage those actions.

The following has been designed with users to reply to this need:
# A new tab "Automatic reminders" should be added in the project model management screen.
# In this tab, the administrator will be able to define as much automatic reminders as need.
# Each automatic reminder has a title.
# For each reminder, the administrator can define the reminder occurence type:
  - "manual": this reminder could be created by the final user. When the final user will click on the "Add" button for a todo reminder in his project, he could choose whether to use the normal todo reminder or this automatic closure reminder type. The final user will set himself the expected date for this reminder.
  - "semi-automatic": the occurence planning of the reminder will be set on the creation of the project. The project creator will be able to choose or not for which dates he wants this automatic closure reminder to be expected. In the project creation popup, he could set several dates for this automatic reminder type.
  - "automatic": in this type, it is the administrator which defines the occurence of the reminder. Several options are possible: (i) a fix frequency (X weeks or X months) after the project creation date, (ii) a fix frequency on calendar dates (every X months at the beginning of the month, etc.)...
# For each automatic closure reminder, the administator will set the fields or project elements to be updated.
  - The fields or project elements which can be used as condition for automatic reminders are: any field of the project model, the progression of the activities, and the indicators (the indicators to be updated cannot be selected by the administrator, but only by the project manager which can attach indicator to automatic reminders to help him or his team to monitor indicators at the required frequency) (the relevance of this last element of the feature (on indicators) should be controlled with a larger panel of users)
  - For each field used as closure condition in the reminder, the administrator will define the earliest valid update date before the reminder expected date for which the field must be updated (ie: 2 weeks before). The reminder will be considered as closed only if every field has its last update date after the earliest valid update date.
# In the todo reminder popup of the reminder, the final user will be able to follow which fields he still has to update to close this reminder
# For a manual reminder, the final user will have the right to change the expected date, but not for the semi-automatic and automatic reminders.
