Summary 0000886: Iterative group aggregation in computation fields
Revision 2017-01-19 12:01 by osarrat
Description When a number field of an iterative group is selected as part of a computation field formula, the aggregation method required to compute a value from each of the iteration of the field should be selected. This aggregation method can be either sum or average. If the computation field is in the same iterative group of one of the number field expected to be part of its computation, another function is also possible to select the value of the field for the current iteration (see child issue 0001038 ).
When using a field part of an iterative group as part of a computation, if no aggregation function is used for that field, the administrator will get the following error message when saving the definition of the field: "Field with field code 'FIELD_CODE' is part of an iterative group. It cannot be part of a computation formula without an aggregation function".
The aggregration method has to be put in the computation formula as functions: "sum()", "average()" and "pick()". The function "pick()" is used to select the value of the number field in the iteration where the computation field is as well. This pick() function is described separately in the issue 0001038.

Examples: "sum(field36) + field12", "pick(field36) / field10"
