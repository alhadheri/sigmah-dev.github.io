Summary0000889: Processable global exports for BI tools
DescriptionRight now, the projects and (upcoming) contacts global exports miss some columns to be easily processed by Business Intelligence (BI) tools.

A few columns or tabs should be added in order to ease this processing, like a permanent id, etc.
The attached Excel files have all requested new columns in red. Columns to be removed like "Budget" are stroked.
Additional InformationA few extra informations to solve some points which were not very clear:
- the new projects Global Export contains: (i) one tab per project model specified to be exported in global export configuration, (ii) one tab for all project links, and (iii) one tab per category containing each the full list of all category items available in that category
- for all list of choice fields linked to a category, an additional column "NAME_OF_FIELD [Category item id]" must be created next the column matching the field: this column will contain a comma-separated list of category item id selected for that project (see attached Global export example)
- for list of choice fields not linked to a category, there is no "NAME_OF_FIELD [Category item id]" additional column to create

 The match between the category column and the category tab is done always manually in the self-service BI tool. So it will be pretty obvious for the user of such a tool to know which column matches to which tab: he just has to look and compare ids or values and see what does match.

But to make things absolutely non-ambiguous, we could have something like:
- column name: "NAME_OF_FIELD (CATEGORY_NAME) [Category item id]", ie: "Thématiques du projet [(Thématique projet)Category item id]"
