Summary0000133: Import and export service for data framework
DescriptionAn import/export service of data frameworks should be created in order to easily transfer data frameworks while sharing a project model across various Sigmah instances.

A solution to deal with the issue of the global nature of a data framework in a Sigmah instance is to link an UUID or GUID with data frameworks on creation.
When a data framework will change (adding a criterion or any other change), its UUID or GUID should change as well.
When a data framework will be transfered, if there is already another data framework with the same UUID or GUID, the data framework won't be imported, and all elements from the models attached to it will be linked to the clone data framework already in database.
