Summary0000409: Delete profiles and privacy groups
DescriptionAs requested in attached user report, a feature to delete profiles and privacy groups is missing.
This feature should be developed with a "Delete" button in the toolbar above the list of profiles, and another one above the list of privacy groups.

When requesting a deletion, the user will get a warning message informing him of the usage of the profiles/privacy groups:
- when attempting to delete a profile, he will get informed of all users still linked to this profile if any
- when attempting to delete a privacy group, he will get informed of all project/orgunit models' fields and profiles linked to this privacy group if any

I agree with aponcon's perception that the following solution would be the best: The admin should unlink them manually in order to make him think about the status and the role of a user for example. When implementing this solution, man have to take care to describe in details to the user each project/orgunit model's field, profile, privacy group attached to the profile/privacy group to be deleted. Because the admin user should not feel frustrated to have to look everywhere to be able remove manually all the links in unknown places.
