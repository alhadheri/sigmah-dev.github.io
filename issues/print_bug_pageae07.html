Summary0000925: Data framework arborescent structure
DescriptionIn order to match expected exporting capabilities, data framework must have an unlimited arborescent structure. This issue describes the new structure to which existing data framework should be migrated.
This structure is based on two types of elements:
- group of elements
- data elements

The arborescent structure is holded by a hierarchy of groups of elements, starting from a common unique root element. Each group of elements has the following characteristics:
- title
- parent group element
- unique or iterative instances
- [optional] indicative type as a free text name to be easily understood
- [optional] documentation URL

The "data elements" are used to represent the content. Each data element has the following characteristics:
- title
- parent group element
- type of data: can be "untyped", or any of the project model basic field type (text field, number field, paragraph field, date field); in addition a regular expression can be specified to constrain even more the content which can be mapped to this element: if such an optional regular expression is specified, an error message must be entered to explain to the user in proper English and not through a regular expression why his value is rejected when he will fill a field mapped to this data element
- [optional] documentation URL

A solution to deal with the issue of the global nature of a data framework in a Sigmah instance is to link an UUID or GUID with data frameworks on creation.
When a data framework will change (adding a group or any other change), its UUID or GUID should change as well.
