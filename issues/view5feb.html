Summary0000593: Draft project model logframe model cleared when derivated draft project logframe modified
DescriptionThe logframe model of a draft project model is cleared once the logframe of a draft project created out of it is modified for the first time.
Steps To Reproduce(i) Create a project model and parameter the logframe tab

(ii) Create a draft project

(iii) Start to enter some information into the logframe

(iv) Go back to the project model logframe tab to change for example the number of SO -> it’s totally empty and doesn’t seem possible to make any change (such as increase the number of OS, indicator…)

(v) It seems to change the log frame configuration you need to duplicate your project model to be able to change log frame configuration.
Additional InformationTemporary solution for the user:
1. create the project model, and make any single modification to the logframe model (and save)
2. create a test project from this model, and update the logframe in it
3. come back to the logframe parameters of the project model => they are cleared.. and now start to put all the parameters you desire for your logframe model
4. create a second test project, and delete the first one
=> the second test project behaves as it should, and its logframe constraints react as they should from the modifications of the project model logframe parameters
