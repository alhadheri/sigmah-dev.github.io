Summary 0000193: Reminder email notification
Revision 2013-10-20 16:56 by osarrat
Description For all "to do" reminders, a new option should give the possibility to send an email notification when creating the reminder.

This notification can have more than one recipient, and can have a comment. The email which will be fired will have:
- subject: the title of the reminder
- sender: author of the reminder
- body: contains the date of the reminder, project/orgunit to which the reminder is attached (code, title and an hyperlink to the project/orgunit on the title) and the optional comment.

When the reminder will be opened for modification, all the details fo the notification sent are accessible (expedition date, author, comment, recipient(s))

If the reminder is created through a "field-linked reminder incentive" (0000534), the name of the field and an hyperlink to this field are included in the notification email.
Revision 2011-05-23 22:59 by osarrat
Description Use cases reported so far for those e-mail alerts:
- when a file has been attached (to be sent to users having rights)
- when a report has been finalised
- when a checbox is checked (the person or profile to who the email should be sent should be define in the model definition)
