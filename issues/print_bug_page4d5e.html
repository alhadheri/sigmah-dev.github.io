Summary0000679: Download of Sigmah javascript application in new language lacks progress information
DescriptionOn v2.0-rc1, when the user requests to open a session in Sigmah with another language than the one he has used previously, Sigmah:
1. loads first with the old language
2. downloads in the background (progress bar visible below the offline icons) the javascript application in the new language
3. when the new language is downloaded, the application is automatically refreshed

This process one weakness: it is confusing for the user because he believes first that its demand to work in another language has been rejected, and then he realizes later on that the application is refreshed in a new language.

A simple solution to inform the user about what's going should be found. For example, a notification message "You have requested to open a session with a new language (_NAME_OF_LANGUAGE_). Sigmah is currently dowloading the user interface in this new language, which will be then displayed after an automatic refresh. You can track the progress of this download in the progress bar below the offline status icons."
