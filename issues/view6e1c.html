Summary0001063: Computation field from linked projects not updated when linked project created
DescriptionOn v2.2-rc2, if you have a computation field with a formula to get information from all funded projects planned budget number field, if you create a new funded project, its initial planned budget won't be taken as part of the computation.

Steps To ReproducePrerequistes :
- project model "source" which has a "planned" number field use as part of the budget ratio.
- project model "target" which has a "sum of planned" computation field use as part of the budget ratio, and this computation field formula is "fundedProjects(source).sum(planned)"

1. create a "target" project
2. from the project management board, create a new funded project by using the "source" project model and give a value different than 0 for the planned budget of this project upon creation
3. go to the details sub-tab of this "target" project
=> your "sum of planned" budget is still set to 0
4. go to the details sub-tab of your "source" project
5. edit the "planned" budget to any value different than 0
6. go back to the details sub-tab of your "target" project
=> now the computation field value has been updated
