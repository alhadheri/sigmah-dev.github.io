Summary0000824: Calendars export to other devices
DescriptionMany users request to have Sigmah calendars available on other devices and programs like smartphone or Thunderbird/Outlook email client.
A first step in that direction would be to provide a read-only export using CalDAV or iCal format. This new feature will add a share link to all calendars which can be used to display them in other programs and devices. By clicking on the "Share link" icon available for each calendar, the user will be able to copy the calendar share URL, to paste it in the program where he wants to use it.

This feature should be developed while keeping in mind that the feature to be able to edit Sigmah calendars remotely will be requested as a second and final step.
Additional InformationJava CalDAV client library : https://github.com/caldav4j/caldav4j [^]
Java iCal library : http://sourceforge.net/projects/ical4j/ [^]


Share link remixed from an icon found on https://www.iconfinder.com/icons/62914/chain_link_seo_tags_icon#size=40 [^] , released with a "Free for commercial use" license by http://www.yummygum.nl/ [^] .

Another way to approach the resolution of this issue is to try to integrate an existing CaldDAV servre Java implementation as part of Sigmah. Calendars are currently under-used and a complete revision of calendar management in Sigmah is an option.
According to this wikipedia page (https://en.wikipedia.org/wiki/Comparison_of_CalDAV_and_CardDAV_implementations#Server_implementations [^] ), there are several CalDAV server java open source implementation. For example, Bedework ( https://www.apereo.org/projects/bedework/bedework-code-repository [^] ) provides a generic CalDAV server Java implementation as part of its codebase.

 Another solution discussed today together would be to limit the scope of this work to the publication of a calendar in ics format through the ical4j library.
To be sure this work will be useful to users, we have to make first a Proof-Of-Concept (POC) able to demonstrate the following scenario :
1. Publish a ics calendar on a URL through the Java POC
2. Subscribe to this URL into a calendar reader (Thunderbird or Outlook)
3. Make the following 4 modifications in the data of the calendar in the Java POC: i) create a new event, ii) delete an avent, iii) modify the time of an event, iv)modifiy the title of an event
4. Checking in the calendar reader if those 4 modifications are visible
