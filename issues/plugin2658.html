[ Details ] [ Diff ] 		Add tomcat in docker-compose file

This tomcat can automatically deploy the sigmah application when packaged
at startup.
It can be started individually by running the following command :
docker-compose up -d tomcat

Just be careful, the sigmah application will be running inside the docker container.
It means the only postgresql database available will be the one from the docker-compose file.
You can retrieve the IP address of the postgresql by running the following command :
docker-compose run postgresql env

This IP address should next be used for configuring the following file :
{project directory}/target/sigmah-2.2-SNAPSHOT/WEB-INF/classes/META-INF/persistence.xml
(file generated after generating the WAR package with Maven)
