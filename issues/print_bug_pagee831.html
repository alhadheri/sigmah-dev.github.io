Summary0000540: Installments follow-up
DescriptionTo be able to follow up installments, a new field type “Installments follow-up” should be available to be added to project or orgunit models.
This feature requires to be able to set the donor currency (unique) for a project.
Each installment is defined by :
- Expected Amount
  o in donor currency
  o In base currency :
    ¤ Computed with a change rate entered manually
    ¤ Amount entered manually
- Received amount
  o in donor currency
  o In base currency:
    ¤ Computed with a change rate entered manually
    ¤ Amount entered manually
- Difference between expected and received amount
  o In donor currency
  o In base currency
- Expected date of reception
- Effective date of reception
- Delay between both dates
- A « Reminder » button which can be transformed into an hyperlink with the title and the date of the reminder linked to this installment
- A comment field

If donor currency is the same as the base currency, the following ‘in donor currency” and “computed with change” fields will be hidden to the end user and filled automatically with 1 for change rate and the same amount as in the base currency for the amounts.
When the installment is created, the expected amount is mandatory. When the effective date of reception if filled, the received amount must have been filled.
If this field is set up for a project or orgunit model, the “received budget” will become read-only and filled automatically as a the total of all received installments.
This new field should be historicized like any other field.
The “Reminder” button will open a popup listing all the existing reminders for the current project/orgunit. The user will have the possibility to select either an existing reminder or create a new one. After this step, the “Reminder” button will be transformed into an hyperlink with the title and the date of the reminder linked to this installment.
Graphically, this will be presented as new component looking like a table filled with rows, each row representing a different installment.
All the content of the installements should be exported through the "Excel export" feature.
