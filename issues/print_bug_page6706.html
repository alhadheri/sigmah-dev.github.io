Summary0000383: Deleting top OrgUnit does nothing
DescriptionIn the organizational unit administration screen, the administrator can select the top orgunit and click on "Delete". After that, he will get a confirmation message to be sure he wants to delete this orgunit. If he clicks on "Yes", he will get a feedback message informing him that this top orgunit has been deleted.

But actually, the top orgunit won't be deleted and no change will be perceived in the application.... which is a good thing ! Because as mentionned in issue 0000382, if a top orgunit is missing, the whole application is blocked for the organization.

To fix this bug of no consequences when clicking on delete for the top orgunit, we suggest to disable the "delete" button for the top orgunit. A tip message should be displayed above the button to inform the user that it is not possible to delete an orgunit having the top position, but it is possible to switch its position with another one and then delete it.
