Summary0000005: GetSchema command needs to be replaced by more fine-grained commands
DescriptionGetSchema and SchemaDTO were designed as a very course-grained DTO to optimize the number of server connections (very time consuming on satellite connections) and enable a high degree of in-memory caching.

However, the course-grain design is very poor in terms of maintaining the code (the queries are very complex) and not scalable beyond a scenario in which a given user has access to 2-3 user databases (original requirement.)

And as we move towards caching data offline in sqlite, there is no compelling reason to maintain the course-grained design.

An analysis of the existing code base needs to be done to see how to break up GetSchema into smaller pieces.

One possibility is to replace with
1/ GetUserDatabases() - and potentially merge with GetProjects
2/ GetDatabaseSchema() - which returns activities/indicators/attributes
