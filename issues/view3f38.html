Summary0001098: UI error when Saving a Phase with modifications in an iterative group
DescriptionOn v2.2-rc5, if you follow the steps to reproduce given below, you will always get an "Unexpected error" while manipulating iterative groups in the end user interface.

The error has been reproduced while running the server both in production or debug mode. And in debug mode, the breakpoint positionned at the beginning of the execute() method of the UpdateProjectHandler was never reached each time I press the Save button. => this error seems to be limited only at the UI level
Steps To Reproduce1. create a draft project model
2. add a Paragraph field in the "Default phase group"
3. add an iterative layout group as second layout group in the "Default root phase"
4. add a Text field in this iterative group
5. create a draft project from this new project model
6. give any value in the Paragraph field
7. create an iteration and give a new to it
8. give any value in the Text field
9. click on "Save modifications"
=> you get the error
